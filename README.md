# One Script for Install 3x-ui+Nginx+certbot
Скрипт полностью автоматизирует установку панели 3X-UI, накатывает официальный Nginx, выбивает SSL-сертификаты и маскирует весь сервер под интерфейс авторизации Cosmos Cloud.

🛠 1. Подготовка перед запуском
Прежде чем запустить скрипт, убедись, что выполнены три простых условия:

Домены: У тебя есть один или несколько доменов (или поддоменов), и их A-записи в DNS уже указывают на IP-адрес твоего сервера.

Чистый порт: Порт 80 должен быть свободен. Если на сервере уже запущен какой-то веб-сервер (например, старый Apache или дефолтный Nginx), его нужно временно остановить.

Права: У тебя есть доступ к серверу под пользователем root или через sudo.

📦 2. Пошаговая установка
Шаг 1: Перенос скрипта на сервер

Создай файл на сервере и откройте его текстовым редактором:

nano setup_mask.sh

Вставь содержимое скрипта, сохрани изменения (Ctrl+O, затем Enter) и выходи (Ctrl+X).

Шаг 2: Выдача прав на исполнение

Сделай скрипт исполняемым:

chmod +x setup_mask.sh

Шаг 3: Запуск

Запусти процесс настройки:

sudo ./setup_mask.sh

🔄 3. Интерактивный процесс (Что нажимать?)

В процессе работы скрипт будет задавать вопросы в консоли. Вот как на них отвечать:

Установка 3X-UI: * Скрипт спросит: Хотите установить/обновить панель 3X-UI прямо сейчас? [Y/n]. Просто жми Enter (выберет «Да»).

Выбор версии: Снова жми Enter, чтобы установилась последняя стабильная версия панели.

🛑 ВАЖНАЯ ПАУЗА! * Скрипт остановится и попросит настроить саму панель.

Открой новое (соседнее) окно терминала, подключись к серверу, введи команду x-ui и обязательно измени внутренний порт (например, 10443) и секретный путь (webBasePath, например, mysecretpanel).

Важно: Внутри самой панели оставляй режим HTTP, шифрованием займется Nginx.

Вернись в основное окно скрипта и нажми Enter для продолжения.

Ввод доменов: * Введи свои домены через пробел. Пример: my-domain.com proxy.my-domain.com. Первый домен автоматически станет основным.

Секретный путь и порт:

Введи тот же путь и порт, которые ты только что указал в консоли панели через команду x-ui.

Email для Let's Encrypt: * Можно ввести свою почту для уведомлений о сертификатах или просто нажать Enter, чтобы пропустить этот шаг.

📂 4. Где что лежит после установки?

/etc/nginx/conf.d/00-maps.conf — глобальные настройки WebSocket карт (чтобы соединения прокси-протоколов не обрывались).

/etc/nginx/conf.d/[твой_главный_домен].conf — основной конфигурационный файл Nginx с проксированием, TLS-хардэнингом и правилами маскировки.

/var/www/html/ — папка с «фейковым» сайтом Cosmos Cloud. Туда же можно закинуть картинку logo.webp, чтобы страница авторизации выглядела идеально.

/etc/letsencrypt/live/[твой_главный_домен]/ — директория с актуальными SSL-сертификатами от Let's Encrypt.

🔒 5. Главные фишки конфигурации

HTTP/2 из коробки: В итоговом конфиге веб-сервера современный протокол HTTP/2 жестко включен через правильную директиву http2 on;. Это гарантирует стабильную скорость и защиту от поломок конфигурации при обновлениях Nginx.

Идеальное алиби для сканеров: Если цензор или автоматический бот попробует просканировать домен, он упрется в чистый фронтенд Cosmos Cloud. При попытке постучаться на скрытые эндпоинты (например, /api/v1/status или отправку формы на /api/v1/auth/login), Nginx сам отдаст им легитимные JSON-ответы-заглушки, даже не дергая саму панель 3X-UI.

Автономное автопродление: В систему прописан деплой-хук. Раз в три месяца Certbot сам обновит SSL-сертификаты и незаметно для пользователей перезагрузит веб-сервер командой systemctl reload, не обрывая текущие активные сессии.

Теперь твоя панель полностью скрыта от посторонних глаз и доступна только по прямому секретному адресу: https://твойдомен.com/твой_путь/
